# 1. Alignment with Assignment Requirements

The assignment specifically states that you are **"responsible for all dataset cleaning,
preprocessing, and ensuring mask alignment"**.
● **The Kaggle Advantage:** CASIA v2.0 has well-documented issues, including 17 images
with resolution misalignments (where the image and mask dimensions are flipped) and
inconsistent naming conventions.
● **Demonstrating Skill:** By downloading the raw data from Kaggle and using a "reusable
script" to fix these issues locally in your notebook, you directly demonstrate the
problem-solving and data-handling skills the evaluators are looking for. If you upload a
pre-cleaned version to Hugging Face, you lose the opportunity to showcase this critical
forensic preprocessing step in your code.

# 2. Technical Performance in Colab

```
● Read Speed: There are documented cases where loading image data via the Hugging
Face datasets library results in significantly slower read speeds (sometimes up to 1000x
slower) compared to standard PyTorch datasets reading from local disk in Colab.
● Disk vs. Memory: CASIA v2.0 is relatively small (~2.6 GB). Downloading it directly to
the Colab instance's disk via the Kaggle API is nearly instantaneous and allows you to
use high-performance DataLoader features like num_workers and pin_memory without
the overhead of the Hugging Face abstraction layer.
```
# 3. Avoiding "Non-Tech Work"

Uploading to Hugging Face requires you to create a Dataset object, map images to masks, and
ensure the repo is public—a process that Tip 3 ("less non-tech work") correctly advises against.
● High-quality versions of CASIA v2.0 already exist on Kaggle (e.g., slug:
divg07/casia-20-image-tampering-detection-dataset).
● Existing Kaggle versions already include the "Ground Truth" masks generated by Nam
Thanh Pham, which is the industry standard for this dataset.

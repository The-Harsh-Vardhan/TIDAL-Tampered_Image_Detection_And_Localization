# 3. Dataset Exploration: Image Tampering Detection Datasets

## 3.1 Overview

Selecting the right dataset is the foundation of any forensic deep learning project. The dataset determines what manipulation types the model can learn, how realistic the training signal is, and whether results are comparable to published benchmarks. This document surveys all major publicly available datasets for image tampering detection and localization.

**Key Requirements from the Assignment**:
- Publicly available dataset(s) with authentic images, tampered images, and **ground truth masks**
- Examples cited: CASIA, COVERAGE, CoMoFoD, or relevant Kaggle datasets
- Must support both **detection** (image-level) and **localization** (pixel-level)

---

## 3.2 Detailed Dataset Profiles

### 3.2.1 CASIA v1.0 (CASIA TIDE v1.0)

| Property | Detail |
|----------|--------|
| **Full Name** | CASIA Image Tampering Evaluation Database v1.0 |
| **Total Images** | ~1,721 (800 Au + 921 Tp) |
| **Forgery Types** | Splicing, Copy-Move |
| **Image Resolution** | Mostly 384×256 (low resolution) |
| **Mask Availability** | Ground truth masks available |
| **Format** | JPEG |
| **Accessibility** | Available on various academic repositories |

**Strengths**: Widely cited in early forensic literature; good for quick prototyping due to small size  
**Weaknesses**: Low resolution images limit forensic artifact preservation; smaller scale limits model generalization; some quality issues with masks  
**Published Benchmarks**: IF-OSN (2022) achieved Pixel-F1 of 68.6%  

**Verdict**: ⚠️ Acceptable as a secondary validation set, but too small and low-resolution for primary training.

---

### 3.2.2 CASIA v2.0 (CASIA TIDE v2.0)

| Property | Detail |
|----------|--------|
| **Full Name** | CASIA Image Tampering Evaluation Database v2.0 |
| **Total Images** | ~4,975 (1,701 Au + 3,274 Tp) |
| **Forgery Types** | Splicing (Tp_D) + Copy-Move (Tp_S) |
| **Image Resolution** | Variable; higher resolution than v1.0 (mostly 240×160 to 900×600) |
| **Mask Availability** | Ground truth masks generated by Nam Thanh Pham (included in Kaggle version) |
| **Format** | JPEG (tampered) + TIFF/BMP (authentic) |
| **Kaggle Slug** | `divg07/casia-20-image-tampering-detection-dataset` |
| **Approximate Size** | ~2.6 GB |

**Naming Convention**:
- `Au`: Authentic images
- `Tp_D_*`: Spliced images (D = "Different" source image)
- `Tp_S_*`: Copy-Move images (S = "Same" source image)
- `*_gt.png`: Corresponding ground truth mask

**Known Issues**:
1. **Resolution Misalignment**: 17 images have mismatched dimensions between the tampered image and its ground truth mask (sometimes width/height are swapped). Must be detected and handled programmatically.
2. **Non-Binary Masks**: Some masks contain intermediate pixel values (not strictly 0 and 255). Require thresholding during preprocessing (e.g., `mask > 128`).
3. **Mixed Formats**: Authentic images are in TIFF/BMP while tampered are in JPEG — need to handle format differences in the data loader.

**Strengths**: Largest standard benchmark for general tampering; includes both splicing AND copy-move; widely used in SOTA papers (enabling direct comparison); easily accessible via Kaggle API  
**Weaknesses**: Moderate resolution; known alignment issues require pre-cleaning; class balance between Au and Tp needs management  

**Published Benchmarks**:
| Model | Year | Pixel-F1 | Pixel-IoU |
|-------|------|----------|-----------|
| VAAS | 2025 | 94.1% | — |
| VASLNet | 2024 | — | 85.1% |
| DAE-Net | 2024 | ~85% | — |
| MVSS-Net++ | 2022 | ~72% | — |

**Verdict**: ✅ **Primary dataset candidate** — best balance of scale, diversity, benchmarkability, and accessibility.

---

### 3.2.3 COVERAGE

| Property | Detail |
|----------|--------|
| **Full Name** | COVERAGE: A Novel Database for Copy-Move Forgery Detection |
| **Total Images** | 200 (100 original + 100 tampered pairs) |
| **Forgery Types** | Copy-Move only |
| **Image Resolution** | Variable (mostly 400×486) |
| **Mask Availability** | 3 mask types: copy source, forged region, paste destination |
| **Unique Design** | Images contain multiple "Similar Genuine Objects" (SGOs) |

**What makes it unique**: COVERAGE is specifically designed to test whether models can distinguish between **naturally occurring similar regions** (e.g., two identical cars in a parking lot) and **actual copy-move forgery**. This "self-similarity ambiguity" makes it one of the hardest benchmarks for copy-move detection.

**Known Issues**:
1. **Resolution Misalignment**: 9 images have dimension mismatches between image and mask
2. Very small dataset (only 100 tampered images) — insufficient for standalone training

**Strengths**: Gold standard for copy-move evaluation; tests the hardest edge case (SGO ambiguity); directly targets bonus criteria  
**Weaknesses**: Too small for primary training; copy-move only (no splicing); requires supplementary training data  

**Published Benchmarks**:
| Model | Year | Pixel-F1 |
|-------|------|----------|
| MMFD-Net | 2024 | 50.4% |
| MVSS-Net++ | 2022 | 48.2% |

**Note**: The low F1 scores reflect the extreme difficulty of this dataset, not model weakness.

**Verdict**: ✅ **Excellent supplementary evaluation set** for copy-move bonus points. Not suitable as primary training data.

---

### 3.2.4 CoMoFoD (Copy-Move Forgery Dataset)

| Property | Detail |
|----------|--------|
| **Full Name** | CoMoFoD: Copy-Move Forgery Detection Dataset |
| **Total Images** | 260 forged image sets |
| **Categories** | Small (512×512) and Large (3000×2000) |
| **Forgery Types** | Copy-Move only |
| **Manipulation Groups** | Translation, Rotation, Scaling, Combination, Distortion |
| **Robustness Variants** | Each image has post-processed versions: JPEG compression, blurring, noise addition, color reduction, brightness change |

**Strengths**: Explicit robustness testing built into the dataset; varied geometric transformations of copy-moved regions; both small and large resolution categories  
**Weaknesses**: Copy-move only; complex dataset structure; less commonly benchmarked in recent SOTA papers; may require custom parsing scripts  

**Verdict**: ⚠️ Valuable for robustness evaluation (bonus criteria) but complex to integrate. Consider for robustness testing if time permits.

---

### 3.2.5 IEEE Forensics (Columbia Image Splicing Dataset)

| Property | Detail |
|----------|--------|
| **Total Images** | ~1,451 (approximately 451 spliced + 1,000 authentic) |
| **Forgery Types** | Splicing only |
| **Mask Availability** | High-quality binary masks |
| **Resolution** | Variable |

**Strengths**: High-quality masks; well-controlled splicing conditions; good for focused splicing evaluation  
**Weaknesses**: Splicing only (no copy-move); smaller scale; less commonly used in recent benchmarks  

**Verdict**: ⚠️ Decent supplementary option for splicing-focused evaluation, but CASIA v2.0 covers splicing more comprehensively.

---

### 3.2.6 NIST Nimble 2016 (NIST16)

| Property | Detail |
|----------|--------|
| **Total Images** | ~564 manipulated images |
| **Forgery Types** | Splicing, Copy-Move, Removal |
| **Mask Availability** | High-quality pixel-level masks |
| **Resolution** | High resolution |
| **Source** | NIST MFC (Media Forensics Challenge) |

**Strengths**: Government-grade dataset with professional manipulation; includes all 3 forgery types; high-quality masks and metadata  
**Weaknesses**: Restricted access (requires agreement/registration); smaller scale; less accessible from Colab  

**Published Benchmarks**:
| Model | Year | Pixel-F1 |
|-------|------|----------|
| DAE-Net | 2024 | 87.1% |

**Verdict**: ⚠️ Excellent quality but accessibility issues may complicate Colab integration.

---

### 3.2.7 IMD2020 (In-the-Wild Manipulations)

| Property | Detail |
|----------|--------|
| **Total Images** | ~2,010 manipulated images |
| **Forgery Types** | Mixed real-world manipulations |
| **Source** | Collected from the internet ("in the wild") |

**Strengths**: Real-world manipulations (not lab-created); tests generalization to unseen forgery styles  
**Weaknesses**: Highly challenging (even SOTA models achieve ~33.8% Pixel-F1); potentially noisy ground truth; not ideal for primary training  

**Published Benchmarks**:
| Model | Year | Pixel-F1 |
|-------|------|----------|
| DAE-Net | 2024 | 33.8% |

**Verdict**: ⚠️ Valuable for out-of-distribution testing but too challenging and noisy for primary training.

## 3.4 Data Availability & Access Strategy

For a Google Colab notebook, data access method matters significantly for reproducibility and performance:

| Access Method | Speed | Reproducibility | Complexity |
|--------------|-------|-----------------|------------|
| **Kaggle API** (`!kaggle datasets download`) | ⚡ Fast (direct to disk) | ✅ Deterministic slug | Low |
| HuggingFace `datasets` | 🐢 Can be slow (documented 1000x slower reads) | ✅ Deterministic | Medium |
| Google Drive mount | ⚡ Fast | ⚠️ Requires manual upload | Low |
| Direct URL download | Variable | ⚠️ URLs may change | Low |

**Recommendation**: Use **Kaggle API** for primary datasets — fastest, most reproducible, and directly demonstrates data engineering skills in the notebook.

## 3.5 Pre-Selection Analysis

Based on the comparison:

1. **For primary training**: CASIA v2.0 is the clear winner — largest scale, both forgery types, widely benchmarked, Kaggle-accessible
2. **For copy-move evaluation (bonus)**: COVERAGE — specifically designed for the hardest copy-move scenario
3. **For robustness testing (bonus)**: CoMoFoD provides pre-built robustness variants, but we can also synthetically apply distortions to CASIA v2.0 test set
4. **Not recommended**: IMD2020 (too noisy), NIST16 (access barrier), CASIA v1.0 (superseded by v2.0)

Detailed justification for the final dataset selection is provided in **Document 04: Best Dataset**.

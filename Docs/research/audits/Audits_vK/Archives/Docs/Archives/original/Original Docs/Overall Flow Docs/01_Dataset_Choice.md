## 1.3 CASIA v2.0 — Complete Profile

### Composition

|----------|--------|-------|-------------|
| Authentic | `Au` | ~1,701 | Unmanipulated camera images (TIFF/BMP format) |
| Spliced | `Tp_D` | ~2,500 | Region from a **different** image pasted in (JPEG) |
| Copy-Move | `Tp_S` | ~757 | Region duplicated **within the same** image (JPEG) |
| **Total** | — | **~4,975** | — |

```
    └── ...
```

### Naming Convention Breakdown
Tampered filenames encode the manipulation type:
```
Tp_D_NRN_S_N_ani00018_sec00096_00138.tif
│  │  │   │ │  │          │        │
│  │  │   │ │  │          │        └── Unique ID
│  │  │   │ │  │          └── Source image category
│  │  │   │ │  └── Target image category
│  │  │   │ └── N = No post-processing
│  │  │   └── S = Same size
│  │  └── NRN = No Rotation/No scaling
│  └── D = Different source (splicing) | S = Same source (copy-move)
└── Tp = Tampered
```

**Image → Mask pairing rule**: mask filename = image stem + `_gt.png`

### Known Issues (Must Handle)

#### Issue 1: Resolution Misalignment (17 images)
17 tampered images have masks where width/height are swapped or different from the image dimensions.

**Detection code** (must include in notebook):
```python
from PIL import Image
import os

misaligned = []
for img_name in os.listdir(tp_dir):
    img_path = os.path.join(tp_dir, img_name)
    mask_name = os.path.splitext(img_name)[0] + '_gt.png'
    mask_path = os.path.join(gt_dir, mask_name)
    
    if os.path.exists(mask_path):
        img = Image.open(img_path)
        mask = Image.open(mask_path)
        if img.size != mask.size:
            misaligned.append(img_name)

print(f"Found {len(misaligned)} misaligned pairs")
```

**Resolution**: Exclude these 17 images. They are only 0.5% of the tampered set — negligible impact on training, but including them would corrupt the ground truth signal.

#### Issue 2: Non-Binary Masks
Masks contain grayscale gradients instead of clean 0/255.

**Resolution**: Binarize during loading:
```python
mask = np.array(Image.open(mask_path).convert('L'))
mask = (mask > 128).astype(np.float32)  # Binary: 0.0 or 1.0
```

#### Issue 3: Mixed Image Formats
Authentic images are TIFF/BMP; tampered images are JPEG/TIF.

**Resolution**: Use `PIL.Image.open()` which handles all formats automatically. Convert to RGB:
```python
image = Image.open(img_path).convert('RGB')
```

## 1.4 COVERAGE — Supplementary Evaluation

### Purpose
Used **only for evaluation** after the model is trained on CASIA v2.0. Tests copy-move detection under the hardest scenario: images with naturally similar objects.

### Composition
- 100 original images + 100 tampered images = 200 total
- Copy-move forgery only
- 3 mask types provided: copy source, forged region, paste destination
- 9 images with resolution misalignment (exclude)

### Usage in Notebook
```python
# After training on CASIA v2.0, evaluate on COVERAGE
coverage_f1 = evaluate(model, coverage_test_loader)
print(f"COVERAGE Pixel-F1: {coverage_f1:.4f}")
# Reference: MVSS-Net++ achieved 48.2%, MMFD-Net achieved 50.4%
```

## 1.5 Why Not Other Datasets?

| Dataset | Reason for Exclusion |
|---------|---------------------|
| CASIA v1.0 | Superseded by v2.0; low resolution; smaller scale |
| CoMoFoD | Complex structure; copy-move only; less commonly benchmarked |
| NIST16 | Restricted access registration; hard to integrate in Colab reproducibly |
| IMD2020 | Too challenging (SOTA = 33.8% F1); noisy ground truth; not suitable for training |
| IEEE Forensics | Splicing only; smaller scale; CASIA v2.0 covers splicing more comprehensively |

### Why Kaggle Over Hugging Face?
1. **Speed**: Direct disk download; no abstraction layer overhead (HF datasets can be up to 1000× slower for image reads)
2. **Skill demonstration**: Raw data download + cleaning script shows data engineering ability the evaluators want to see
3. **Simplicity**: One command; no Dataset object creation; no repo management
4. **Precedent**: The Kaggle version already includes the standard ground truth masks by Nam Thanh Pham

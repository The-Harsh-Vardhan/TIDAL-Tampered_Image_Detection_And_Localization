### 4.2.2 Alignment with Assignment Requirements

The assignment explicitly states:
> *"Use one or more publicly available datasets containing original (authentic) images, tampered images, and ground truth masks for localization. Examples include the CASIA Image Tampering Dataset..."*

CASIA v2.0 is the **first example cited** by the evaluators. Using it demonstrates direct alignment with their expectations and enables them to immediately contextualize the results against known benchmarks.

### 4.2.3 Benchmarkability

CASIA v2.0 is the **most commonly reported** benchmark in forensic papers. This allows:
- Direct comparison of our results against published SOTA numbers
- Evaluators can immediately assess whether our results are reasonable
- Published SOTA range gives us a target: Pixel-F1 of 70-85% is competitive; 60-70% is solid for a 1-week project

### 4.2.4 Why Not Use Hugging Face?

While Hugging Face provides excellent dataset hosting, for this project Kaggle is superior because:
1. **Speed**: Documented cases of HuggingFace `datasets` library being up to 1000x slower for image reads compared to local disk access in Colab
2. **Assignment alignment**: The assignment requires demonstrating data cleaning and preprocessing skills — downloading raw data from Kaggle and fixing known issues showcases this
3. **Simplicity**: Single download command; no need to create Dataset objects or manage HF abstractions
4. **Existing versions**: High-quality CASIA v2.0 on Kaggle already includes the ground truth masks generated by Nam Thanh Pham (industry standard for this dataset)

## 4.3 COVERAGE as Supplementary Evaluation

### 4.3.1 Why Include COVERAGE?

The assignment's bonus criteria specifically mentions:
> *"Successfully detecting subtle tampering such as copy-move manipulation or splicing from similar textures."*

COVERAGE is **designed exactly for this scenario**. Its "Similar Genuine Objects" (SGO) design means images naturally contain objects that look similar to each other — the model must distinguish between natural repetitions and actual copy-move forgery. Strong performance on COVERAGE is direct evidence of the bonus criteria.

### 4.3.2 How to Use COVERAGE

- **Not for training**: Only 100 tampered images — too small
- **For evaluation only**: After training on CASIA v2.0, evaluate on COVERAGE to demonstrate generalization to hard copy-move cases
- **Benchmarkable**: Can compare against MMFD-Net (50.4% F1) and MVSS-Net++ (48.2% F1)

## 4.4 Data Cleaning Requirements

### 4.4.1 CASIA v2.0 Cleaning Pipeline

These are **known issues** that must be handled programmatically in the notebook:

#### Issue 1: Resolution Misalignment (17 images)
**Problem**: 17 tampered images have ground truth masks where the width and height are swapped (e.g., image is 384×256 but mask is 256×384).

**Solution**: Write a validation function that checks `image.size == mask.size` for every pair. For misaligned pairs, either:
- **Option A**: Resize the mask to match the image (introduces minor interpolation artifacts)
- **Option B**: Exclude these 17 images from training (cleaner; 17 out of ~3,274 tampered is negligible)

**Recommendation**: Option B (exclude) — preserves mask integrity and 17 images is only 0.5% of the tampered set.

#### Issue 2: Non-Binary Masks
**Problem**: Ground truth masks contain intermediate pixel values (grayscale gradients) instead of strict 0/255 binary.

**Solution**: Apply threshold during preprocessing:
```
mask = (mask > 128).astype(np.uint8) * 255
```
This is standard practice in all published forensic papers using CASIA.

#### Issue 3: Image-Mask Pairing Convention
**Problem**: Images and masks are in separate directories with specific naming conventions.

**Solution**: Implement a pairing function:
- Tampered images: `Tp/Tp_D_NRN_nat00025_11037.jpg`
- Corresponding mask: `Gt/Tp_D_NRN_nat00025_11037_gt.png`
- Pattern: mask filename = image stem + `_gt` + `.png`

#### Issue 4: Authentic Images Have No Masks
**Problem**: Authentic images don't have masks (entire image is authentic = mask of all zeros).

**Solution**: Generate all-zero masks dynamically for authentic images during data loading:
```
if is_authentic:
    mask = np.zeros((height, width), dtype=np.uint8)
```

### 4.4.2 COVERAGE Cleaning Pipeline

**Issue**: 9 images have resolution misalignment (same type as CASIA).  
**Solution**: Same approach — exclude misaligned pairs during evaluation.

## 4.5 Data Split Strategy

### 4.5.1 Split Ratios

| Split | Ratio | Purpose |
|-------|-------|---------|
| Training | 85% | Model learning |
| Validation | 7.5% | Hyperparameter tuning, early stopping, checkpoint selection |
| Test | 7.5% | Final evaluation (never seen during training or tuning) |

### 4.5.2 Stratification Requirements

The split must be **stratified** to maintain the ratio of:
1. Authentic vs. Tampered images across all splits
2. Splicing (Tp_D) vs. Copy-Move (Tp_S) within the tampered set

This prevents the model from seeing disproportionately more of one class during training.

### 4.5.3 Class Balancing Strategy

**Raw distribution**: ~1,701 Au + ~3,274 Tp (approximately 1:2 ratio)

**Training subset approach**:
- Use ~1,000 authentic images + ~2,000 tampered images for training
- This maintains the natural 1:2 ratio while keeping the dataset manageable for T4 GPU training time
- Alternatively, use all data but apply class-weighted sampling in the DataLoader

## 4.6 Augmentation Strategy

Using `albumentations` for synchronized image-mask augmentation:

| Augmentation | Purpose | Parameters |
|-------------|---------|------------|
| HorizontalFlip | Orientation invariance | p=0.5 |
| VerticalFlip | Orientation invariance | p=0.5 |
| RandomRotate90 | Rotation invariance | p=0.5 |
| RandomBrightnessContrast | Lighting robustness | brightness_limit=0.2, contrast_limit=0.2 |
| GaussNoise | Noise robustness (bonus) | var_limit=(10, 50) |
| ImageCompression | JPEG robustness (bonus) | quality_lower=50, quality_upper=100 |
| Resize | Standardize input | height=512, width=512 |
| Normalize | Network input normalization | ImageNet mean/std |

**Critical**: `albumentations` automatically applies the same spatial transforms to both image and mask, ensuring they remain aligned. This is why it's preferred over `torchvision.transforms` for segmentation tasks.
